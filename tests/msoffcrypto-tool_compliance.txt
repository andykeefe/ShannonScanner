NONCOMPLIANT CRYPTOGRAPHIC MATERIAL: SHA1
   - REASON: SHA1 is an approved security function for FIPS 140-2 according to FIPS 180-4.
     - However, the NIST notes that it is vulnerable to collision attacks.
     - Therefore, NIST recommends that SHA-1 be disallowed for security operations.
     - For FIPS 140-3, SHA-1 may be used for integrity checks, but not for digital signature generation.
   - COMPLIANCE RECOMMENDATION: Use SHA-2 or SHA-3 for FIPS 140-2/140-3 compliance.
   - AFFECTED FILES:
         - msoffcrypto/method/rc4_cryptoapi.py
         - msoffcrypto/method/ecma376_agile.py
         - msoffcrypto/method/ecma376_standard.py
